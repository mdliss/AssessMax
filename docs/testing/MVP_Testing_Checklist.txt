AssessMax MVP Testing and Validation Checklist
==============================================

Version: 1.0
Last Updated: 2024-01-15
Status: Ready for Testing

Overview
--------
This document provides a comprehensive testing checklist for the AssessMax MVP
to validate core functionality, educator workflows, system resilience, and security.

Testing Phases
--------------
1. Core Functionality Testing
2. Educator User Flow Validation
3. System Resilience Assessment
4. Security Testing
5. Issue Resolution and Retesting

═══════════════════════════════════════════════════════════════════════════
PHASE 1: CORE FUNCTIONALITY TESTING
═══════════════════════════════════════════════════════════════════════════

Authentication and Authorization
---------------------------------
[ ] User registration with email/password
[ ] User login with valid credentials
[ ] User login fails with invalid credentials
[ ] MFA enrollment and verification
[ ] Password reset flow
[ ] Session persistence across browser refreshes
[ ] Session timeout after 30 minutes of inactivity
[ ] Automatic logout on session expiration
[ ] Role assignment (educator, admin, read_only)
[ ] RBAC: Educator can access assigned classes
[ ] RBAC: Admin can access all features
[ ] RBAC: Read-only cannot modify data
[ ] Token refresh mechanism
[ ] Logout clears all session data

Transcript Ingestion
--------------------
[ ] Upload JSONL transcript successfully
[ ] Upload CSV transcript successfully
[ ] Upload TXT transcript successfully
[ ] Reject unsupported file formats
[ ] Validate required metadata (class_id, date, roster)
[ ] Handle missing metadata gracefully
[ ] Store file in S3 with correct path
[ ] Create DynamoDB metadata record
[ ] Generate unique job ID
[ ] Duplicate upload detection (idempotent)
[ ] File size validation (max 50MB)
[ ] Handle concurrent uploads
[ ] Error messages are clear and actionable

Artifact Ingestion
------------------
[ ] Generate presigned URL for upload
[ ] Upload PDF artifact via presigned URL
[ ] Upload DOCX artifact via presigned URL
[ ] Upload PNG image via presigned URL
[ ] Upload JPG image via presigned URL
[ ] Presigned URL expires after configured time
[ ] Reject files exceeding size limit
[ ] Validate artifact metadata
[ ] Link artifact to student record
[ ] Store artifact in S3 with versioning
[ ] Create artifact metadata in DynamoDB

NLP Scoring Pipeline
--------------------
[ ] Detect transcript language correctly
[ ] Clean and normalize transcript text
[ ] Segment text into sentences accurately
[ ] Map speakers to text segments (diarization)
[ ] Detect empathy skill cues
[ ] Detect adaptability skill cues
[ ] Detect collaboration skill cues
[ ] Detect communication skill cues
[ ] Detect self_regulation skill cues
[ ] Generate skill scores (0-100 scale)
[ ] Calculate confidence scores (0-1 scale)
[ ] Scores are reproducible with same input
[ ] Handle multilingual transcripts
[ ] Handle transcripts with errors/noise
[ ] Pipeline completes within SLA (5 min for 1000 utterances)

Evidence Extraction
-------------------
[ ] Extract text spans for each skill
[ ] Rank evidence by relevance
[ ] Provide rationale for each evidence span
[ ] Cite transcript line numbers correctly
[ ] Cite artifact page numbers correctly
[ ] Evidence spans contain meaningful content
[ ] Top 3 evidence spans per skill are accurate
[ ] Evidence resolves to source location (≥95%)

Assessment API
--------------
[ ] GET /v1/assessments/{student_id} returns latest assessment
[ ] Latest assessment includes all skill scores
[ ] Latest assessment includes confidence scores
[ ] GET /v1/assessments/{student_id}/history returns paginated history
[ ] History is ordered by date (most recent first)
[ ] Pagination works correctly (limit/offset)
[ ] 404 for non-existent student
[ ] 401 without authentication
[ ] Response time < 500ms for single assessment
[ ] Response time < 2s for history (100 records)

Class Dashboard API
-------------------
[ ] GET /v1/classes/{class_id}/dashboard returns class metrics
[ ] Class averages calculated correctly
[ ] Student count is accurate
[ ] Total assessments count is accurate
[ ] Date range calculated correctly
[ ] Student summaries include latest assessments
[ ] Student summaries include assessment counts
[ ] Student summaries include average scores
[ ] 404 for non-existent class
[ ] Response time < 3s for class of 30 students

Evidence API
------------
[ ] GET /v1/evidence/{assessment_id} returns evidence spans
[ ] Evidence grouped by skill
[ ] Evidence includes span text
[ ] Evidence includes span location
[ ] Evidence includes rationale
[ ] Evidence includes score contribution
[ ] 404 for non-existent assessment
[ ] Response time < 1s for assessment with 50 evidence spans

Streamlit Dashboard
-------------------
[ ] Dashboard loads without errors
[ ] Login page displays correctly
[ ] Login redirects to class overview after authentication
[ ] Class Overview: Displays class metrics
[ ] Class Overview: Shows student summary table
[ ] Class Overview: Radar chart renders correctly
[ ] Class Overview: Export to CSV works
[ ] Class Overview: Export to PDF works
[ ] Student Detail: Displays latest assessment

[ ] Student Detail: Export to CSV works
[ ] Student Detail: Export to PDF works
[ ] Trends page loads (placeholder OK)
[ ] Uploads page loads (placeholder OK)
[ ] Navigation between pages works smoothly
[ ] Refresh button updates data
[ ] Loading indicators display during API calls
[ ] Error messages display for failed API calls

Batch Processing
----------------
[ ] Lambda function processes normalization job
[ ] Lambda function processes scoring job
[ ] SQS queues messages correctly
[ ] SQS triggers Lambda on message arrival
[ ] Step Functions orchestrates workflow
[ ] Step Functions handles task failures
[ ] Step Functions retries failed tasks (max 3)
[ ] DLQ captures permanently failed messages
[ ] Job status updates in DynamoDB
[ ] Job logs available in CloudWatch

Job Status API
--------------

[ ] Job status shows current processing state

[ ] Admin can list all jobs

[ ] Pagination works for job list

Data Persistence
----------------
[ ] PostgreSQL stores assessment records

[ ] Alembic migrations apply successfully
[ ] Database indexes improve query performance

[ ] DynamoDB stores job metadata

[ ] DynamoDB TTL deletes expired records
[ ] S3 versioning enabled for all buckets
[ ] S3 objects encrypted at rest (KMS)
[ ] S3 lifecycle policies transition to IA/Glacier

PHASE 3: SYSTEM RESILIENCE ASSESSMENT
═══════════════════════════════════════════════════════════════════════════

Load Testing
------------
[ ] System handles 50 concurrent users
[ ] System handles 8 class-days per hour
[ ] System handles 30 students per class
[ ] System handles 3000 utterances per transcript
[ ] API response times remain acceptable under load
[ ] No timeouts occur under normal load
[ ] Database connection pool doesn't exhaust
[ ] Lambda concurrency limits not reached
[ ] SQS queues don't back up
[ ] CloudWatch metrics show healthy state

Failure Scenarios
-----------------
[ ] Database connection failure handled gracefully
[ ] S3 upload failure returns clear error
[ ] Lambda timeout triggers retry
[ ] SQS message processing failure moves to DLQ
[ ] Step Functions circuit breaker prevents cascading failures
[ ] API returns 503 when backend unavailable
[ ] Dashboard shows friendly error for API failures
[ ] Partial transcript processing doesn't corrupt data
[ ] Invalid file upload doesn't crash API
[ ] Malformed transcript handled without data loss

Recovery Testing
----------------
[ ] Database connection recovers after network blip
[ ] Failed jobs can be retried manually
[ ] System state remains consistent after failures
[ ] No orphaned records in database
[ ] No orphaned files in S3
[ ] DynamoDB eventually consistent reads work correctly
[ ] Backup and restore procedures work

Scalability Testing
-------------------
[ ] System scales to handle 100 classes
[ ] System scales to handle 3000 students
[ ] System scales to handle 10000 assessments
[ ] Database queries remain fast with large datasets
[ ] S3 handles 10000+ objects per bucket
[ ] DynamoDB handles high read/write throughput
[ ] Lambda functions scale automatically
[ ] No bottlenecks in processing pipeline

PHASE 4: SECURITY TESTING
═══════════════════════════════════════════════════════════════════════════

Authentication Security
-----------------------
[ ] Passwords hashed with bcrypt/Argon2
[ ] Sessions use secure, HTTP-only cookies
[ ] CSRF protection enabled
[ ] Session tokens expire after timeout
[ ] Failed login attempts rate limited
[ ] Account lockout after 5 failed attempts
[ ] Password reset tokens expire after 1 hour
[ ] MFA codes expire after 5 minutes
[ ] No credentials in URL parameters
[ ] No credentials in logs

Authorization Security
----------------------
[ ] Users can only access own classes
[ ] Educators cannot access admin endpoints
[ ] Read-only users cannot modify data
[ ] Role escalation attempts blocked
[ ] Direct object reference protection (student IDs)
[ ] API endpoints validate user permissions
[ ] Database RLS policies enforced (if applicable)

Input Validation
----------------
[ ] SQL injection attempts blocked
[ ] XSS attempts sanitized
[ ] Path traversal attempts blocked
[ ] File upload restrictions enforced
[ ] File type validation using magic bytes
[ ] File size limits enforced
[ ] Input length limits enforced
[ ] Special characters handled safely

API Security
------------
[ ] All endpoints require authentication
[ ] HTTPS enforced (no HTTP)
[ ] TLS 1.3 minimum
[ ] CORS configured correctly
[ ] Rate limiting enabled
[ ] Request size limits enforced
[ ] No sensitive data in error messages
[ ] API versioning prevents breaking changes

Data Security
-------------
[ ] PII encrypted at rest (S3, RDS, DynamoDB)
[ ] PII encrypted in transit (TLS)
[ ] Database connections encrypted
[ ] Backups encrypted
[ ] KMS keys rotated regularly
[ ] Access logs don't contain PII
[ ] Audit logs capture all data access
[ ] Data deletion is permanent

Infrastructure Security
-----------------------
[ ] VPC isolation for databases
[ ] Security groups follow least privilege
[ ] IAM roles follow least privilege
[ ] No hardcoded credentials in code
[ ] Secrets stored in AWS Secrets Manager
[ ] Lambda environment variables encrypted
[ ] S3 buckets block public access
[ ] CloudWatch logs encrypted
[ ] WAF rules protect against common attacks

Compliance
----------
[ ] FERPA requirements met
[ ] PPRA requirements met
[ ] Parental consent process documented
[ ] Data retention policies implemented
[ ] Data deletion procedures documented
[ ] Audit trail complete
[ ] Privacy policy published
[ ] Terms of service published

Penetration Testing
-------------------
[ ] Automated vulnerability scanning (Trivy/Snyk)
[ ] Manual penetration testing completed
[ ] OWASP Top 10 vulnerabilities addressed
[ ] No P1 security issues found
[ ] P2 security issues have remediation plan
[ ] Security findings documented
[ ] Remediation verified
