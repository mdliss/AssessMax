1. Legal Framework
------------------

### FERPA (20 U.S.C. ยง 1232g; 34 CFR Part 99)

FERPA protects the privacy of student education records. Key requirements:

- Schools must obtain written consent before disclosing PII from education records
- Parents/eligible students have right to inspect and review education records
- Parents/eligible students have right to request amendments to records
- Schools must maintain records of disclosures

### PPRA (20 U.S.C. ยง 1232h; 34 CFR Part 98)

PPRA protects student privacy in surveys and assessments. Key requirements:

- Prior written consent required for certain surveys
- Parents have right to inspect surveys and instructional materials
- Opportunity to opt out of certain activities
- Protection of personal information collected from students

- Email address
- Class enrollment

- Teacher observations
- Assessment timestamps

**Tier 3 - Non-Sensitive:**
- Anonymized research data
- System logs (no PII)

- Behavioral observations
- Social-emotional skill assessments
- Communication patterns

3. Access Controls
------------------

### Role-Based Access Control (RBAC)

**Administrator Role:**
- Full system access
- User management
- Configuration changes
- Audit log access

**Educator Role:**
- View students in assigned classes
- View and generate assessments
- Access class aggregated data
- Export reports for assigned classes

**Read-Only Role:**
- View class aggregated data only
- No access to individual student records
- No export capabilities

### Technical Controls

1. AWS Cognito authentication with MFA
2. Session tokens expire after 60 minutes
3. Automatic logout after 30 minutes of inactivity
4. IP-based access restrictions (configurable)
5. Failed login attempt monitoring
6. Principle of least privilege for all roles

### Access Logging

All access to education records is logged with:
- User ID and role
- Timestamp
- Action performed
- Record(s) accessed
- IP address
- Session ID

4. Consent and Authorization
-----------------------------

### Parental Consent Requirements

Prior to using AssessMax, schools must:

1. Provide written notice to parents about:
   - Type of data collected
   - Purpose of collection
   - How data will be used
   - Third parties with access
   - Data retention period
   - Security measures

2. Obtain written consent for:
   - Collection of student data
   - Assessment participation
   - Data retention beyond academic year

### Consent Management

- Consent forms stored securely
- Consent status tracked per student
- Annual consent renewal required
- Easy opt-out process available
- Consent revocation honored within 5 business days

### Data Sharing Agreements

Third-party access requires:
- Written data sharing agreement
- FERPA compliance attestation
- Security standards certification
- Annual compliance review
- Immediate termination clause

5. Data Retention and Deletion
-------------------------------

### Retention Periods

**Active Records:**
- Student assessment data: Retained while student is enrolled
- Historical data: 2 years after last enrollment
- Audit logs: 3 years minimum

**Deleted Records:**
- Soft delete initially (30-day recovery period)
- Hard delete after 30 days
- Backup purging after 90 days

### Data Deletion Procedures

**Upon Request:**
1. Verify identity of requester
2. Confirm authorization to request deletion
3. Process deletion within 10 business days
4. Provide confirmation to requester
5. Document deletion in audit log

**Automated Deletion:**
- Students inactive for 2 years: flagged for review
- Reviewed records deleted if no retention requirement
- Anonymized data retained for research (if consented)

### Right to Request Deletion

Parents/eligible students may request:
- Full account deletion
- Specific record deletion
- Data export before deletion
- Confirmation of deletion

6. Security Measures
--------------------

### Data Encryption

**In Transit:**
- TLS 1.3 for all API communications
- HTTPS only (no HTTP)
- Certificate pinning for mobile apps
- VPN required for admin access

**At Rest:**
- AES-256 encryption for S3 buckets
- KMS encryption for all data stores
- Encrypted database backups
- Encrypted log files

### Infrastructure Security

1. **Network Security:**
   - VPC isolation
   - Private subnets for databases
   - Security groups with least privilege
   - WAF for API protection
   - DDoS protection via CloudFront

2. **Access Security:**
   - Multi-factor authentication required
   - Regular password rotation
   - Strong password requirements (12+ chars, complexity)
   - Session management and timeout
   - IP whitelist for admin access

3. **Database Security:**
   - Encrypted connections only
   - Parameterized queries (SQL injection prevention)
   - Database user least privilege
   - Regular security patches
   - Automated backups with encryption

4. **Application Security:**
   - Input validation and sanitization
   - Output encoding
   - CSRF protection
   - XSS prevention
   - Regular security scans

### Third-Party Security

All third-party services must:
- Be FERPA compliant
- Undergo security assessment
- Sign data processing agreement
- Provide SOC 2 Type II certification
- Submit to annual audit

8. Breach Response
------------------

### Incident Response Plan

**Detection:**
- 24/7 monitoring of security events
- Automated alerting for suspicious activity
- Regular log review
- User-reported incidents

**Containment:**
1. Isolate affected systems (within 1 hour)
2. Preserve evidence
3. Assess scope of breach
4. Document all actions

**Investigation:**
1. Identify breach cause
2. Determine data accessed/exfiltrated
3. Identify affected individuals
4. Document timeline

**Notification:**

If breach affects >100 individuals:
- School district: Within 24 hours
- Department of Education: Within 72 hours
- Affected individuals: Without unreasonable delay
- Media (if required): As per state law

**Remediation:**
1. Patch vulnerabilities
2. Update security controls
3. Reset compromised credentials
4. Enhance monitoring
5. Conduct lessons learned

### Breach Prevention

- Regular security training
- Penetration testing
- Vulnerability scanning
- Security patch management
- Incident response drills

Appendices

### Appendix A: Definitions

**Education Records:** Records directly related to a student maintained by an
educational agency or institution or by a party acting for the agency or institution.

**Directory Information:** Information contained in an education record that
would not generally be considered harmful or an invasion of privacy if disclosed.

**PII (Personally Identifiable Information):** Information that can be used to
distinguish or trace an individual's identity.

**Eligible Student:** A student who has reached 18 years of age or is attending
a postsecondary institution.
